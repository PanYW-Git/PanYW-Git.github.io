常用DOS命令（持续更新）

发表于2023-04-10|更新于2023-09-06|DOS命令
|字数总计:3.8k|阅读时长:13分钟|阅读量:
命令 作用 备注
Linux
java -jar –spring.profiles.active=prod java启动 –spring.profiles.active=prod 启动配置类为pord属性的
基础命令
ls [-a -l -h] [Linux路径] 列出目录下的路径 -a 列出所有文件（包括隐藏文件） -l 竖向查看 -h 需要和-l一起使用，显示文件大小带上单位 ls -l权限 一共10个曹位 1：-表示文件，d表示文件夹，l表示软连接 24：所属用户权限 57：所属用户组权限 8~10：其他用户权限 r 读 w 写 x 执行
pwd 打印当前目录
cd ./ 表示当前目录 ../ 返回上一级 ../../返回上两级 ~/ 表示Home目录
clear 清屏
mkdir [Linux路径]ll 创建文件夹 -p 创建多层级目录
touch [路径] 创建文件
cat [路径] 查看文件，将内容全部显示出来不支持翻页
more [路径] 查看文件，支持翻页（空格：翻下一页，b:上一页，q：退出）
cp [-r] 参数1 参数2 复制文件 -r 可选 用于复制文件夹使用，采用递归 参数1，被复制的文件 参数2，复制去的文件
mv [被移动路径] [移动去路径] 移动文件/文件夹
rm [-r -f] [参数1，参数2，参数3……] 删除文件/文件夹 -r用于删除文件夹，递归的方式删除 -f表示强制删除（不会弹出提示确认信息） 参数：可以删除多个文件/文件夹 支持通配符test或test或test匹配任意内容
which [要查找的命令名称] 命令程序文件存放在哪儿（只针对命令）
find 起始路径 -name “被查找文件名” 从起始路径开始按名称查找文件 例：起始路径为/表示全盘搜索 -name 名称搜索（支持通配符）
find 起始路径 -size +|-n[Kmg] 从起始路径开始按文件大小查找文件 -size 按文件大小搜索 +、-表示大于或小于 [Kmg]表示大小单位，k(小写)表示kb，M表示MB，G表示GB 例：查找大于1G的文件：find / -size +1G
grep [-n] 关键字 文件路径 从文件中通过关键字过滤文件行 -n 可选，在结果中显示匹配行的行号 关键字 必填 过滤的关键字 文件路径 必填 表示要过滤的内容文件路径
wc [-c -m -l -w] 文件路径 统计文件的行数、单词书等 -c 统计bytes数量 -m 统计字符数量 -l 统计行数 -w 统计单词数量
管道符| 将管道符 左边命令的结果，作为右边命令的输入
echo “输出的内容” echo命令在命令行内输出指定内容
反引号` 用反引号包围的内容将作为命令执行，而非普通字符 例：echo pwd
重定向符> >> 类似于sout >将左侧命令的结果，【覆盖】写入到符号右侧指定的文件中 >>将左侧命令的结果，【追加】写入到符号右侧指定的文件中
tail [-f -num] 文件路径 -f，表示持续跟踪 -num,表示，查看尾部多少行 注：num表示数字不是关键字
vi\vim文本编辑器 linux文本编辑器 命令模式： i 进入输入模式 yyp 复制一行 dd 删除一行 u 撤销 crtl r 反向撤销 / 进入搜索模式 n 向下继续搜索 N 向上继续搜索 : 进入底线命令模式 输入模式：正常输入 按esc 退回命令模式 底线命令模式： :wq 保存并退出 :q! 强制退出 :set nu 显示行号 :set paste 设置粘贴模式（用于从外部复制内容过来，保证格式不错乱）
su [-] [用户名] 切换用户 -符号可选，表示是否在切换用户后加载环境变量 切换用户后可以exit命令退回上一个用户
sudo 命令 以普通用户执行root管理员命令
visudo 为普通用户配置sudo认证 进入编辑器后在最后一行添加 用户名 ALL=(ALL) NOPASSWORD: ALL
groupadd [用户组名称] 创建用户组
groupdel [用户组名称] 删除用户组
useradd [-g -d] 用户名 -g 选填、指定用户的组，不指定-g，会创建同名祖 -d 选填、指定用户Home路径，不指定默认/home/用户名
userdel [-r] -r 可选、是你出用户的Home目录，不使用则保留Home数据
id [用户名] 查看用户信息
usermod -aG [用户组] [用户名] 将指定用户加入只当用户组
getent password 查看当前系统中有哪些用户
getent group 查看当前系统中有哪些用户组
chmod [-R] 权限 文件或文件夹 修改文件、文件夹权限 -R 可选、对文件夹内的全部内容应用同样的操作 例： chmod u=rwx,g=rx,o=x hello.txt.将文件权限修改为：rwxr-x–x chmod 775 hello.txt 将文件权限修改为rwxrwxr-x
chown [-R] [用户][:][用户组] 文件/文件夹 修改文件、文件夹所属的用户和用户组 注：普通用户无法修改，只适用于root用户执行 -R 可选、对文件夹内的全部内容应用同样的操作 例： chown root hello.txt 用户修改为root chown :root hello.txt 用户组修改为root
history 查看历史输入的命令 可以和grep连用
yum [-y] [install | remove | search] 可选，-y.,自动确认，无需手动确认安装或卸载过程 install：安装 remove：卸载 search：搜索
systemctl [start | stop | status | enable | disable] 对Linux的服务进行控制（启动关闭查看状态，开启开机自启，关闭开机自启）
ln -s [被连接的文件或文件夹] [连接去的地址] 创建软连接，可将文件、文件夹链接到其他位置 类似windows中的快捷方式
date [-d] “[+格式化字符串]” date “+%Y-%m-%d %H:%M:%S” 查看系统时间 -d 按照给定的字符串显示日期，一般用于日期计算 格式化字符串 %Y 年 %y 年份后两位数字 %m 月份（0112） %d 日（0131） %H 消失（0023） %M 分钟（0059） %S 秒（00，60） %s 自1970-01-01 00：00：00 UTC到现在的秒数
rm -f /etc/localtime sudo -ln -s /usr/share/zoneinfo/Asia/Shanghai /etc/localtime 切换Linux时区 1、删除现在时区 2、添加需要设置的时区的软连接到/etc/localtime
ntpdate -u ntp.aliyun.com ntp手动校准 默认自动校准
hostname 获取主机名
hostname -set-hostname 主机名 修改主机名
ping [-c num] ip或主机名 -c 可选，检查次数，不用-c将无限次持续监测
wget [-b] url 文件下载器（类似迅雷等下载软件），将网址拷贝过来就可以直接下载到服务器 -b 可选，后台下载，并将日志放到当前工作目录的wget-log文件
curl [-O] url 可以发送http网络请求，可用于：下载文件、获取信息 -O 可选,可以下载文件，如果不加只发送请求
curl cip.cc 获取公网ip地址
nmap [地址] 查看所有端口被占用的情况 注：nmap不是自带需要yum -y install nmap
netstat -anp|grep 端口号/进程号 查看指定端口占用情况 注：此命令不是自带需要yum -y install net-tools
ps [-e -f] ps -ef|grep 需要查找的信息（可以的查进程号，启动时间启动用户等） 查看进程（类似windows中的任务管理器） -e 可选，显示全部进程 -f 可选，展示全部信息
kill [-9] 进程号 杀死进程 -9 表示强制关闭进程，不使用此项则要求关闭，是否关闭看京城自身的处理机制
top 查看CPU，内存的使用情况，类似windows的任务管理器 -p 只显示某个进程信息 -d 设置刷新时间，默认5s -c 显示产生晶晨固定完整命令，默认是进程名 -n 指定刷新次数，比如 top -n 3，刷新3此后退出 -b 以非交互全屏模式运行，以批次的形式显示 -i不显示任何闲置（idle）或无用（zombie）的进程 -u 查找特定用户启动的进程
df -h 磁盘相关信息查看
iostat [-x] 磁盘使用情况监控 -x 可选，详情信息
sar -n DEV 查看网络的相关统计 -n 查看网络，DEV表示查看网络接口 此命令使用方式很多后续其他需要可以百度
env 查看当前系统中记录的环境变量
${环境变量名} 获取环境变量的值
export 变量名 = 值 临时设置用户环境变量
vi ~/.bashrc 永久设置用户环境变量，当前用户的home文件下编辑~/.bashrc写入export 变量名 = 值 注：保存后需要使用生效命令生效
source ~/bashrc 文件生效，生效后可使用环境变量
vim /etc/profile 永久设置全局环境变量，写入export 变量名 = 值 注：保存后需要使用生效命令生效
tar [-c -v -x -f -z -C] 参数1 参数2…..参数N tar/tar.gz压缩解压命令 压缩常用 tar -cvf 压缩文件包名.tar 需要压缩的文件1 文件2 文件3…… tar -zcvf 压缩文件包名.tar.gz 需要压缩的文件1 文件2 文件3…. 解压常用 tar -xvf test.jar 将文件解压至当前目录 tar -xvf test.tar -C /home 解压test.tar,将文件解压至指定目录/home tar -zxvf test.tar.gz 以Gzip模式解压test.tar.gz
zip [-r] 参数1 参数2…参数N zip压缩文件 参数1 压缩文件名 参数2~参数N 压缩的文件 -r如果有文件夹使用，表示包含路径下的所有文件
unzip [-d] 文件名 zip解压 -d，指定解压去的路径 例 unzip test.zip -d /home 解压test.zip到/home路径
systemctl start firewalld 开启防火墙
firewall-cmd –zone=public –add-port=8100/tcp –permanent 开放指定端口
firewall-cmd –reload 重启防火墙
systemctl disable firewalld 禁止防火墙开机自启
快捷键
ctrl+C 强制停止
ctrl+L 清屏
ctrl+D 退出账号登录或特定程序如python
![命令前缀] 搜索历史中最近的命令前缀与本次相符的执行 注：如果相隔太久就不要用这个命令！
ctrl+r 搜索历史命令，如果需要使用按回车，不需要删除
ctrl+a 跳到命令开头
ctrl+e 跳到命令结尾
ctrl+键盘左右键 跳到前/后的一个单词
Nginx
nginx -t 检查配置文件的正确性
nginx 启动nginx服务
nginx -s stop 关闭nginx服务
nginx -s reload 重启nginx服务
Docker
systemctl start docker 启动docker服务
systemctl stop docker 停止docker服务
systemctl restart docker 重启docker服务
docker –help 查看docker所有命令帮助文档
docker 命令 –help 查看指定命令帮助文档
docker pull 镜像名称:版本号 拉取指定镜像，版本号不写默认最新
docker images 查看镜像
docker save -o 文件名 镜像名:版本号 将docker的指定镜像导出到指定路径 版本号可以写lastetst 最后版本
docker load -i 镜像压缩包名称 将镜像压缩包导入到docker中
docker rmi 镜像名:版本号 删除指定镜像
docker run 运行容器 注：要去dockerhub官方查看每个程序的启动方式，都是不一样的
docker run –name nginx -p 80:80 -d nginx 启动nginx –name：给容器起一个名字 some-nginx -p：将linux服务器端口与容器端口映射，左边是宿主机端口，右边是容器端口 8080：linux服务器端口 80：docker映射端口 表示用户访问8080就是访问nginx -d 后台运行容器 nginx：镜像名称，例如nginx
docker pause 容器名 指定容器暂停（挂起）内存不释放，cpu不执行了
docker unpause 容器名 指定容器解除暂停（挂起）
docker stop 容器名 指定容器停止
docker start 容器名 指定容器开始
docker rm [-f] 删除指定容器 删除指定容器（包括本地文件一起删除） 可选，-f：强制删除运行中的容器
docker ps [-a] 查看正在运行的容器：默认只显示运行中的容器 可选，-a:查看所有容器
docker logs 容器名 查看容器日志
docker exec -it 容器名 bash 不推荐使用此方法修改 进入容器内部执行一个命令 -it:给当前进入的容器创建一个标准输入，输出终端，允许与容器交互 bash 进入容器后执行的命令，bash是一个linux终端交互命令
Redis
systemctl start redis 启动redis 注：这一步需要把redis注册为服务
systemctl stop redis 停止redis
systemctl enable redis 设置为开机自启
redis-cli -h redis地址 -p 端口号 -a redis密码 进入redis命令行客户端
ping 与redis服务端做心跳测试，服务端正常会返回pong
keys [key名称，查全部使用*] 查看符合模板的所有key 不建议在生产环境上使用，redis是单线程容易造成阻塞
del [key1,key2,key3……] 删除一个key或者多个key
exists key 判断key是否存在，不存在返回0
expire key secends 设置一个key的有效期，到达有效期后key会从内存中被移除返回值-2 key：键 secends：有效期（秒）
ttl 键名称 查看KEY的剩余时间 返回值： 正数：剩余存活秒数 -1：永久有效 -2：被移出内存
setnx key value 存储一个键值对（键必须不存在的时候才会成功插入）
setex key secends value 存储一个键值对，并设置存活秒数 secends：存活时间（秒）
get key 获取一个键
mset [key1,value1] []key2,value2] [key3,value3] 存储多个键值对
mget key1 key2 key3…… 获取多个键
incr key 让整型的key自增1 类似于i++
incrby key 增长数 让整型的key增长自定义数
incrbyfloat key 增长数 让浮点型的key增长自定义数
Hash类型
hset 项目名:业务名:类型:id key field value 使用hash的方式存储数据 key:键 field:字段 value:值
hget key field 获取使用hash的方式存储的数据
hgetall key 获取使用hash的方式存储key下的所有数据
List类型
lpush key list类型左边插入一个或多个元素
rpush key list类型右边插入一个或多个元素
Set类型
sadd key 向set中添加一个或者多个元素
srem key 移除set中的指定元素
smembers 获取set中的所有元素
SortedSet类型
windows
netstat -ano | findstr 5050 查看端口号pid
taskkill /pid 8992 -f 通过pid杀死进程
shutdown -s -t 时间（秒） 多少秒关闭电脑
shutdown -a 取消关机
calc 打开计算器
notepad 打开记事本
Hexo
hexo s -p 8850 启动hexo 端口号为8850
hexo clean 清除hexo缓存 更新博客
hexo g hexo生成静态文件
hexo d hexo上传到github page
